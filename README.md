## 产品定位

在 Todoist、TickTick、Things 等成熟产品的包围下，任务管理工具正变得越来越复杂——文件夹、看板、标签、优先级矩阵……功能堆叠让工具的认知成本越来越高。

**「绪」** 反其道而行，回归"一张纸一支笔"般的直觉体验：

- **极简**：无文件夹、无看板、无项目——只有待办列表
- **高效**：自然语言智能解析日期，随想随记
- **聚焦**：每日焦点推荐，自动排序重要事项
- **流畅**：Apple 级微交互，让操作成为一种享受

### 核心理念

> 最好的任务管理工具，是让你感觉不到它的存在。

## 核心功能

### 核心待办 CRUD

新建、编辑、标记完成/撤销、删除——所有基本操作一气呵成。

- **顶部固定输入框**，类 iOS 备忘录风格的快捷创建面板
- **点击条目进入详情页**，修改字段后自动保存
- **圆形复选框 + 完成动画**：对勾填充 → 文字渐灰 → 删除线 → 任务下沉

### 自然语言日期解析

智能识别中文日期表述，无需手动选择日历：

```
明天下午3点跟设计评审   → 📅 2026-06-11 15:00
大后天上午开会          → 📅 2026-06-13 09:00
下周一提交周报          → 📅 2026-06-16 00:00
每天晚上10点复盘        → 🔄 每日 22:00
```

**支持格式：** 相对日期（今天/明天/后天）、星期（周一~周日）、具体日期（X月X日）、重复模式（每天/每周/每工作日）

### 每日焦点推荐

每天首次打开应用，自动从今日待办中智能推荐 **1-3 条焦点任务**。

```
┌─────────────────────────────────────┐
│  ★ 今日焦点                         │
│    ● 完成产品评审            [确认] │
│    ● 撰写周报                [确认] │
└─────────────────────────────────────┘
```

- **排序算法：** 高优先级 > 中优先级 > 低优先级 → 重要标记优先 → 创建时间优先
- **交互反馈：** 可确认采纳或刷新换一批，焦点任务在列表中持久置顶

### 跨设备同步（V1）

基于 Supabase + PostgreSQL 的云端同步方案：

- **乐观更新**：操作后立即更新本地 UI，无需等待服务器确认
- **离线支持**：断网时操作存入 IndexedDB，联网后自动同步
- **冲突解决**：版本号乐观锁 + 最后写入优先（LWW）策略
- **重试机制**：指数退避（1s → 2s → 4s → 8s，最多 5 次）

## 交互体验

### 信息架构

「绪」采用**极简的两层信息架构**，你永远只需要专注三件事：

> **查看 → 新建 → 完成**，全部在 **3 步以内**完成。

```
┌─────────────────────────────────────┐
│          待办列表（首页）              │
│  ┌─────────────────────────────────┐ │
│  │ ★ 今日焦点                      │ │
│  │  ├─ ✅ 完成产品评审             │ │
│  │  └─ ⬜ 撰写周报                  │ │
│  ├─────────────────────────────────┤ │
│  │ 今天                             │ │
│  │  ├─ ⬜ 明天下午3点跟设计评审      │ │
│  │  └─ ⬜ 提交代码 review           │ │
│  ├─────────────────────────────────┤ │
│  │ 明天                             │ │
```

### 微交互设计

| 交互 | 体验 |
|------|------|
| ✅ **任务完成动画** | 对勾填充（100ms）→ 文字渐灰（200ms）→ 删除线（100ms）→ 下沉（300ms） |
| 🗑 **删除确认** | 移动端左滑露出红色删除按钮，桌面端悬停显示菜单 |
| 🎯 **空状态引导** | 首次使用"欢迎使用绪"、全部完成"太棒了"、无今日待办"放松一下" |
| ⏳ **加载状态** | 骨架屏动画（800ms-1500ms），杜绝白屏 |

## 技术架构

### 推荐技术栈

```
┌─────────────────────────────────────────────────────────┐
│                    客户端 (Web PWA)                      │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │  React 组件  │  │  Zustand 状态  │  │  IndexedDB    │  │
│  │  (UI层)      │  │  (状态管理)   │  │  (离线缓存)    │  │
│  └──────┬──────┘  └──────┬───────┘  └───────┬───────┘  │
│         └────────────────┼──────────────────┘           │
│                          │                              │
│                    API 请求层 (fetch/axios)               │
└──────────────────────────┼──────────────────────────────┘
                           │ HTTPS / WebSocket
┌──────────────────────────┼──────────────────────────────┐
│                   Supabase                               │
│  ┌───────────────────────┼──────────────────────────┐   │
│  │       Supabase Auth (认证)                        │   │
│  ├───────────────────────┴──────────────────────────┤   │
│  │     Supabase Realtime (实时推送)                   │   │
│  ├──────────────────────────────────────────────────┤   │
│  │              PostgreSQL 数据库                     │   │
│  │  tables: users, todos, focuses, sync_logs        │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| **前端框架** | Next.js (React) | SSR/SSG 支持，优秀的开发体验 |
| **状态管理** | Zustand | 轻量、TypeScript 友好 |
| **后端/BaaS** | Supabase | 开箱即用的 Auth + 数据库 + Realtime |
| **数据库** | PostgreSQL | 成熟稳定，Supabase 原生支持 |
| **离线缓存** | IndexedDB | 浏览器原生离线存储 |
| **部署** | Vercel | 与 Next.js 深度集成 |

### 数据模型

```sql
-- 用户表
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT UNIQUE NOT NULL,
  name        TEXT,
  avatar_url  TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 待办表
CREATE TABLE todos (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title        TEXT NOT NULL,
  notes        TEXT DEFAULT '',
  due_date     TIMESTAMPTZ,
  is_completed BOOLEAN DEFAULT FALSE,
  is_focus     BOOLEAN DEFAULT FALSE,
  priority     TEXT CHECK (priority IN ('low','medium','high')) DEFAULT 'medium',
  sort_order   INTEGER DEFAULT 0,
  is_deleted   BOOLEAN DEFAULT FALSE,
  version      INTEGER DEFAULT 1,
  created_at   TIMESTAMPTZ DEFAULT NOW(),
  updated_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_todos_user_id ON todos(user_id);
CREATE INDEX idx_todos_due_date ON todos(due_date);
CREATE INDEX idx_todos_user_completed ON todos(user_id, is_completed);
```

### V1.0 - 启程（2026年7月）

**包含功能：**
- [x] 待办 CRUD（新建/编辑/完成/删除）
- [x] 自然语言日期解析

**明确不包含：**

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/your-org/xu.git
cd xu

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.local
# 编辑 .env.local，填入你的 Supabase 项目信息

# 启动开发服务器
npm run dev
```

| 变量名 | 说明 |
|--------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目 URL |
